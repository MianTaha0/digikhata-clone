# Invoicing / Bills (Phase 2c) — Design Spec

**Date:** 2026-04-21
**Project:** `/Users/macbookpro/Documents/digikhata-clone`
**Depends on:** Phase 2b Expense Tracker (Room v3)
**Goal:** Add a per-book invoicing module — create, view, edit, and share invoices as PDFs — wired to the existing "Bills" bottom-nav tab (currently a Coming Soon placeholder).

---

## Overview

Invoicing is a standalone per-book module. Each invoice has a customer (picked from the existing Clients table), multiple line items with per-line tax, an invoice-level discount, optional due date and notes, and a payment status derived from `amountPaid` vs. `grandTotal`. Invoice numbers are auto-incremented per book with a user-configurable prefix (default `INV-`).

Invoicing does NOT auto-sync with the customer ledger. Users manage ledger credits separately if they want them.

PDFs are generated by a new `InvoicePdfGenerator` utility using Android's built-in `android.graphics.pdf.PdfDocument` (zero external dependencies, same pattern as the Kharcha Book report PDF). Sharing supports Download (system chooser) and Send-via-WhatsApp.

Line-item names autocomplete from the 20 most recent item names used on this book — no Inventory dependency.

## Decisions (from brainstorming)

| # | Decision |
|---|---|
| Q1 | Line items: free text + autocomplete from past items |
| Q2 | Per-line tax % (each row has its own tax rate) |
| Q3 | Invoice-level discount (amount or %) |
| Q4 | Three payment states: Paid / Partial / Pending (single `amountPaid`) |
| Q5 | Independent stream — no auto-sync with customer ledger |
| Q6 | Auto-increment per book + custom prefix (default `INV-`) |
| Q7 | Canvas-drawn PDF via `android.graphics.pdf.PdfDocument` |
| Q8 | Fields: invoice# + issueDate + customer + dueDate + notes |
| Q9 | Delivery: Download PDF + Send via WhatsApp |

### Derived values — `InvoiceTotals` and `InvoiceStatus`

File: `app/src/main/java/com/digikhata/domain/model/InvoiceTotals.kt`

```kotlin
enum class InvoiceStatus { PAID, PARTIAL, PENDING }

data class InvoiceTotals(
    val subtotal: Double,
    val totalTax: Double,
    val discountAmount: Double,
    val grandTotal: Double,
    val balance: Double,
    val status: InvoiceStatus
)

data class InvoiceWithItems(val invoice: Invoice, val items: List<InvoiceItem>)
```

Pure computation utility — `util/InvoiceCalc.kt`:

```kotlin
object InvoiceCalc {
    fun compute(invoice: Invoice, items: List<InvoiceItem>): InvoiceTotals {
        val subtotal = items.sumOf { it.quantity * it.unitPrice }
        val totalTax = items.sumOf { it.quantity * it.unitPrice * it.taxPercent / 100.0 }
        val discount = if (invoice.discountIsPercent)
            subtotal * invoice.discountValue.coerceAtMost(100.0) / 100.0
        else
            invoice.discountValue.coerceAtMost(subtotal)
        val grand = (subtotal + totalTax - discount).coerceAtLeast(0.0)
        val balance = (grand - invoice.amountPaid).coerceAtLeast(0.0)
        val status = when {
            invoice.amountPaid >= grand - 0.001 -> InvoiceStatus.PAID
            invoice.amountPaid > 0.0 -> InvoiceStatus.PARTIAL
            else -> InvoiceStatus.PENDING
        }
        return InvoiceTotals(subtotal, totalTax, discount, grand, balance, status)
    }

    fun displayNumber(prefix: String, sequence: Int): String =
        "$prefix${sequence.toString().padStart(4, '0')}"
}
```

### `saveInvoice` behavior

Must run inside a single Room transaction (annotate the repo method with `@Transaction` on a helper DAO method, or wrap in `db.withTransaction { ... }`).

Pseudocode:
```
val now = System.currentTimeMillis()
val id = if (inv.id == 0L) {
    val seq = invoiceDao.nextSequenceNumber(inv.businessId)
    val newInv = inv.copy(sequenceNumber = seq, createdAt = now, updatedAt = now)
    invoiceDao.insertInvoice(newInv)     // returns new id
} else {
    invoiceDao.updateInvoice(inv.copy(updatedAt = now))
    invoiceItemDao.deleteByInvoice(inv.id)
    inv.id
}
invoiceItemDao.insertAll(items.mapIndexed { idx, it -> it.copy(invoiceId = id, sortOrder = idx) })
return id
```

### `recordPayment` behavior

```
val inv = invoiceDao.getById(id).first() ?: return
val items = invoiceItemDao.getByInvoice(id).first()
val totals = InvoiceCalc.compute(inv, items)
val newPaid = (inv.amountPaid + amount).coerceAtMost(totals.grandTotal)
invoiceDao.updateInvoice(inv.copy(amountPaid = newPaid, updatedAt = System.currentTimeMillis()))
```

`deleteInvoice` relies on FK CASCADE for items. No image files involved.

### Form validation rules

- Customer required — Save disabled until picked
- At least one line item required with `quantity > 0` and `unitPrice >= 0`
- `taxPercent` clamped 0–100
- If `discountIsPercent`, `discountValue` clamped 0–100; else clamped 0 to subtotal
- Numeric fields parse on blur; show the parsed form

## PDF Generation

### Utility — `util/InvoicePdfGenerator.kt`

```kotlin
object InvoicePdfGenerator {
    fun generate(
        context: Context,
        business: Business,
        customer: Client,
        invoice: Invoice,
        items: List<InvoiceItem>,
        totals: InvoiceTotals
    ): Uri?
}
```

- Same Canvas-drawn approach as the Kharcha Book `PdfGenerator`. Zero new dependencies.
- Page size A4 (595 × 842pt). Margin 40pt.
- Red header band (height 100pt) in `#E74425` with white text: business name (titleLarge), phone/address (small), "INVOICE" label (top-right), displayNumber, issue date, due date (if set).
- **BILL TO** block below the header: customer name (bold), phone (if set), address (if set).
- Items table with columns: `#`, `ITEM`, `QTY`, `RATE`, `TAX%`, `TOTAL`. Alternating row shading. Truncate long item names with ellipsis.
- Right-aligned totals block at bottom-right: Subtotal, Tax, Discount (labeled with "Rs X" or "X%" as appropriate), GRAND TOTAL (bold, larger), Paid, BALANCE. Paid/Balance rows shown only if `amountPaid > 0`.
- Status watermark: if PAID, draw a faded green "PAID" diagonally across the items area (alpha ~50, 60pt font, rotated −20°).
- Page-break aware: when items overflow, start a new page with repeated column headers. Footer on the last page only: "Generated by DigiKhata • {timestamp}".
- Output file saved to `getExternalFilesDir(DIRECTORY_DOCUMENTS)/DigiKhata_${displayNumber}.pdf`.
- Returns a Uri via `FileProvider.getUriForFile(context, "${applicationId}.provider", file)`. FileProvider is already declared in Phase 1.

### Download button

`InvoiceDetailScreen.kt` — "Download PDF" button:
1. `viewModel.generatePdfUri(context)` on `Dispatchers.IO` → returns Uri.
2. Fire `Intent.ACTION_VIEW` with `setDataAndType(uri, "application/pdf")` and `FLAG_GRANT_READ_URI_PERMISSION`.
3. Wrap with `Intent.createChooser(...)` for fallback.

### Send via WhatsApp button

Same PDF generation, then:
```kotlin
val send = Intent(Intent.ACTION_SEND).apply {
    type = "application/pdf"
    putExtra(Intent.EXTRA_STREAM, pdfUri)
    putExtra(Intent.EXTRA_TEXT, "Invoice $displayNumber — Total ${CurrencyUtils.format(totals.grandTotal, business.currency)}")
    addFlags(Intent.FLAG_GRANT_READ_URI_PERMISSION)
    setPackage("com.whatsapp")
}
try {
    context.startActivity(send)
} catch (_: ActivityNotFoundException) {
    context.startActivity(Intent.createChooser(send.also { it.setPackage(null) }, "Share invoice"))
}
```

## What Phase 2c Does NOT Include

- Inventory integration (line items stay free-text; autocomplete pulls from history)
- Logo rendering on PDF (business entity has `logoLocalPath` from Phase 1 but unused here; reserved space in header)
- Multiple invoice templates / themes
- Email and Print delivery (WhatsApp + Download only)
- Payment history with multiple rows (single `amountPaid` field)
- Auto-sync to customer ledger on payment
- Recurring invoices
- Invoice duplication action
- Currency override per invoice
- Tax-inclusive pricing toggle

These are candidates for future 2c.x follow-ups.

## Success Criteria

- Bottom-nav "Bills" tab opens `InvoiceListScreen` (no longer ComingSoon).
- Creating an invoice works: pick a customer, add ≥ 1 line item (each with qty/price/tax%), set invoice-level discount, due date, notes, save. Appears in list with auto-number `INV-0001`, `INV-0002`, etc.
- Line-item name field shows autocomplete dropdown listing recent item names (up to 20) from this book.
- Detail screen renders with computed totals matching the create-form preview.
- Record Payment adds to `amountPaid`; status badge transitions Pending → Partial → Paid as payments are entered.
- Download PDF produces a readable `DigiKhata_INV-XXXX.pdf` with all fields rendered.
- Send WhatsApp opens WhatsApp (if installed) with the PDF attached and preview text; falls back to a system chooser otherwise.
- Editing an invoice replaces line items atomically.
- Deleting an invoice removes items (CASCADE) but not the customer.
- Switching active book hides invoices from other books.
- Renaming `invoicePrefix` in a future BookSettings update would only change the display; stored `sequenceNumber` is stable.
- `./gradlew assembleDebug` → BUILD SUCCESSFUL.
- Room v3 → v4 migration runs cleanly (no data loss).
